1. Struktur Folder

Seluruh aplikasi disusun dengan prinsip separation of concerns — setiap jenis berkas memiliki rumahnya sendiri agar mudah ditemukan dan dipelihara.

LeksaLaw/ │ ├── index.html ← Dashboard utama (halaman pembuka) ├── nilai.html ← Halaman riwayat nilai & progress ├── favorit.html ← Halaman daftar quiz favorit ├── tentang.html ← Halaman tentang aplikasi ├── README.html ← Dokumentasi ini │ ├── assets/ │ ├── css/ │ │ ├── app.css ← Style global (sidebar, dashboard, kartu, dll) │ │ └── quiz.css ← Style khusus halaman quiz │ ├── js/ │ │ ├── app.js ← Logic dashboard, sidebar, pencarian, tema │ │ ├── quiz-engine.js ← Mesin quiz (skor, navigasi, sesi) │ │ └── build-quiz.js ← (opsional) generator HTML quiz │ ├── img/ ← Tempat logo/gambar │ └── icons/ ← Tempat ikon kustom │ ├── data/ │ └── config.js ← ⭐ FILE UTAMA — daftar semua mata kuliah & quiz │ └── quizzes/ ├── BIK_quiz1.html ├── HDAGK_quiz1.html ├── HIPA_quiz1.html ├── TIPISUS_quiz1.html ├── TIPISUS_quiz2.html ├── TIPISUS_quiz3.html ├── HPidIn_quiz1.html ├── Perbankan_quiz1.html ├── AKP_quiz1.html └── AKP_quiz2.html ... (dan seterusnya)
💡 Catatan desain: Folder per-semester (Semester I, II, dst.) digantikan dengan satu folder quizzes/ yang flat, karena pengelompokan semester dan mata kuliah sudah ditangani sepenuhnya oleh data/config.js. Pendekatan ini menghindari duplikasi struktur dan membuat pencarian file lebih cepat — terutama saat koleksi quiz sudah mencapai ratusan file.
2. Cara Menambah Mata Kuliah Baru

Setiap mata kuliah didaftarkan sebagai satu objek di dalam array COURSES pada file data/config.js.

  1. Buka file data/config.js
  2. Tambahkan objek baru ke dalam array COURSES dengan format di bawah
  3. Simpan file — Dashboard akan otomatis menampilkan mata kuliah baru tanpa perlu mengubah file lain
{ id: "HUKACP", // kode unik, huruf kapital, tanpa spasi slug: "hukum-acara-perdata", // untuk URL (opsional dipakai) name: "Hukum Acara Perdata", semester: 7, // nomor semester code: "HKUM4302", // kode mata kuliah resmi color: "#2563eb", // warna aksen kartu (hex) icon: "⚖️", // emoji ikon description: "Deskripsi singkat...", quizzes: [] // lihat bagian 3 untuk mengisi ini }
⚠️ Pastikan nilai id tidak sama dengan mata kuliah lain yang sudah ada — ini digunakan sebagai kunci penyimpanan progress.
3. Cara Menambah Quiz Baru
  1. Salin salah satu file di folder quizzes/ sebagai titik awal (misalnya BIK_quiz1.html)
  2. Ubah nama file sesuai konvensi: {ID_MATAKULIAH}_quiz{N}.html, contoh: HUKACP_quiz1.html
  3. Di dalam file, cari blok const QUIZ_DATA = {...} di bagian paling bawah
  4. Ubah quizId, courseId, courseName, title, dan array questions sesuai kebutuhan
  5. Daftarkan quiz baru ini ke dalam array quizzes milik mata kuliah terkait di data/config.js
quizzes: [ { id: "HUKACP_Q1", // harus unik di seluruh aplikasi title: "Quiz 1 — Gugatan & Jawaban", subtitle: "60 soal · Tingkat Sedang", totalQuestions: 60, level: "sedang", // "sedang" | "menengah" | "advance" file: "quizzes/HUKACP_quiz1.html", createdAt: "2026-08-01" } ]
4. Format Data Soal

Setiap quiz menyimpan soalnya di dalam array questions pada bagian QUIZ_DATA. Format setiap soal:

{ topic: "Nama Topik / Modul", // tampil sebagai tag di atas soal q: "Teks pertanyaan di sini...", opts: ["Pilihan A", "Pilihan B", "Pilihan C", "Pilihan D"], correct: 2, // index 0-based dari opsi yang benar (di atas = "Pilihan C") source: "Modul 3, Hal. 45", // opsional, referensi sumber explanation: "Penjelasan tambahan..." // opsional, tampil di Mode Belajar }
FieldWajib?Keterangan
topicTidakLabel kategori soal, tampil sebagai badge
qYaTeks pertanyaan (boleh mengandung HTML seperti <b>, <em>)
optsYaArray pilihan jawaban, minimal 2 opsi
correctYaIndex (mulai dari 0) jawaban yang benar
sourceTidakReferensi sumber/halaman modul
explanationTidakPenjelasan tambahan jawaban
5. Mengganti Logo & Warna
Mengganti Logo

Logo saat ini menggunakan emoji (⚖️) sebagai placeholder ringan tanpa file gambar. Untuk mengganti dengan gambar:

  1. Letakkan file logo (PNG/SVG) di assets/img/logo.png
  2. Cari elemen <div class="sidebar-logo">⚖️</div> di setiap halaman
  3. Ganti dengan <img src="assets/img/logo.png" style="width:36px;height:36px;border-radius:10px">
Mengganti Warna Tema

Seluruh palet warna terpusat di bagian atas assets/css/app.css dalam blok :root:

:root { --navy: #0f2557; // warna sidebar & header gelap --blue: #2563eb; // warna aksen utama (tombol, link) --teal: #0891b2; // aksen sekunder }

Cukup ubah nilai hex pada variabel tersebut — seluruh komponen di aplikasi akan otomatis menyesuaikan.

6. Cara Kerja Local Storage

Aplikasi menyimpan data pengguna di Local Storage browser (tidak memerlukan server/database). Berikut kunci yang digunakan:

KunciIsi
leksalaw_progressRiwayat nilai setiap quiz yang pernah dikerjakan
leksalaw_favoritesDaftar ID quiz yang ditandai favorit
leksalaw_themePreferensi tema (light/dark)
sidebar_collapsedStatus sidebar (terbuka/tertutup)
leksalaw_session_{quizId}Sesi quiz yang belum selesai (auto-resume)
💡 Data ini tersimpan per-browser dan per-perangkat. Jika dipindah ke flashdisk dan dibuka di komputer lain, riwayat nilai tidak akan ikut terbawa kecuali menggunakan fitur Backup di bawah.
7. Backup & Restore Data

Untuk membackup progress belajar secara manual, buka Developer Console browser (tekan F12), lalu jalankan:

Backup (Export)
console.log(JSON.stringify({ progress: localStorage.getItem('leksalaw_progress'), favorites: localStorage.getItem('leksalaw_favorites') }));

Salin hasil output dan simpan ke file teks sebagai cadangan.

Restore (Import)
const backup = /* tempel JSON hasil backup di sini */; localStorage.setItem('leksalaw_progress', backup.progress); localStorage.setItem('leksalaw_favorites', backup.favorites); location.reload();
8. Troubleshooting
MasalahSolusi
Mata kuliah baru tidak muncul di DashboardPastikan tidak ada kesalahan sintaks (koma/kurung) di data/config.js; cek Developer Console (F12) untuk error
Quiz menampilkan halaman kosongPeriksa apakah path file pada config sesuai dengan nama file di folder quizzes/
Nilai tidak tersimpanPastikan browser tidak dalam mode Incognito/Private — mode ini menghapus Local Storage saat ditutup
Tampilan rusak setelah edit CSSPeriksa kembali tanda kurung kurawal { } pada file CSS yang diedit
